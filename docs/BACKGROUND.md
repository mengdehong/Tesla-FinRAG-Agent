## Python开发工程师（大模型后端方向）   实习招聘任务

构建一个特斯拉跨年财报智能问答系统上市公司财报（如10-K年报和10-Q季报）包含海量信息，回答一个复杂财务问题往往需要同时检索和整合来自多个年份、多个文档（甚至不同类型报表）的数据。例如，“对比2021年至2023年，特斯拉在哪个季度的汽车毛利率最高？当时的宏观背景在管理层讨论中是如何描述的？”

需要构建一个系统，能够处理特斯拉2021、2022、2023、2024、2025年（如有）的所有10-K年报及10-Q季报，并准确回答涉及跨年趋势、多文档对比、数字计算与文本关联的复杂问题。
任务描述
1. 环境管理
使用 uv 初始化项目，所有依赖通过 uv add 添加，生成 uv.lock 文件。
依赖建议：pypdf / pdfplumber、requests、chromadb / faiss-cpu、gradio / streamlit、openai / z.ai
当然也可以使用你喜欢的方式构建环境，如conda/pip
2. 数据处理与索引
  文档解析：需要将PDF财报中的文本段落和表格数据都提取出来，并保留其来源信息（如文件名、页数、表格标题）。对于表格，建议转换为结构化格式（如Markdown或JSON）以便后续检索和理解。
  - 分块策略：由于问题复杂性，单纯按长度切分文本效果有限。你建议考虑一种更智能的分块方式，例如基于文档原有章节标题（如“流动性”、“汽车毛利率”）进行语义分块，并确保每个表格作为一个独立且完整的单元。请在报告中说明你的最终策略及其理由。
  - 向量化：使用免费Embedding API（阿里云DashScope或HuggingFace Inference API）或者本地运行轻量Embedding模型，将所有文本块和表格描述转换为向量，存入向量数据库。 
3. 检索与生成
  混合检索增强：实现结合关键词（BM25）和向量的混合检索，尤其要确保包含特定财务术语（如“Free Cash Flow”）或年份（如“2022 Q3”）的查询能精准命中。
  - 复杂问题回答流程：设计一个能够处理多步推理的问答流程。例如，对于“2022年哪份季报提到了供应链挑战，并且该季度的营收环比变化如何？”这类问题，系统可能需要：
    1. 先检索包含“供应链挑战”的文档和段落。
    2. 再定位到同一季度的财务数据表。
    3. 最后整合信息生成答案。
4. 高阶测评与失败评估
  构建高阶测试集：你需要设计至少5个真正复杂的问题，如：
      *   跨越多份文档：例如，比较2021年10-K和2023年10-K中关于“中国市场风险”的描述变化。
      *   需要数字计算：例如，计算2022年四个季度“研发费用”的总和，并与2021年全年对比。
      *   关联文本与表格：例如，找到“汽车利润率”最低的季度，并提取该季度“管理层讨论与分析”部分的关键句子。
      *   隐含时间顺序：例如，描述“从2021年到2023年，特斯拉的自由现金流是如何在每个季度波动的？哪份文件首次提到了某个特定工厂的产能瓶颈？”
  - 深入失败案例分析：系统性地运行你的测试集，并重点分析至少5个回答失败或质量不佳的案例。对于每个案例，你需要详细探究：
    - 失败表象：答案是错误的、不完整的，还是引用了错误来源？
    - 溯源排查：是检索阶段遗漏了关键表格？是模型误解了跨文档的数据关系？还是PDF解析时数字格式出错？
    - 根本原因：最终定位到的系统瓶颈是什么？（例如，分块破坏了表格的完整性、无法关联同一季度的文本和表格、缺乏多步推理能力）
    - 具体改进方案：提出针对该原因的、可落地的改进措施（例如，引入基于文档结构的检索、为表格创建单独的索引并添加时间戳元数据、设计Agent流程先定位相关文档再提取数据）。
5. 交互与报告
  使用Streamlit 构建界面，允许用户选择查询范围（如“仅10-K”或“所有文档”），输入问题，并展示答案、引用片段和（可选的）检索调试信息。
